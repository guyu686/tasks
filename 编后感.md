# question

- 你构想了一个什么项目，最终完全实现你的构想了吗？
- 你在项目中，是如何用 HTML 给它搭建骨骼、用 CSS 给它增添颜值、用JS给它赋予变化的？
- 我们总是在关关难过关关过中成长，说说你在项目中遇到的最大难题是啥？你又是如何克服和解决的？
- 你认为当前的项目还需要优化和补完吗？如果将来你成为一方 dalao，你会想回到最初为这个项目填上最后一块砖吗？

# 编后感

## 构想项目

* 由于刚刚经历了大学的第一个运动会，我们计算机学院拿到了总分第一的好成绩。碰巧我在学校公众号上看到了介绍此次计算机学院校运会情况的推文，想着我能不能做一个类似的网页来祝贺一下计算机学院夺得桂冠。
* 只能说最终我的构想只实现了一部分，在决定做这个网页的时候，我一直在想各种创意来突出我的网页，就像我github的自我介绍一样，增添各种动图，光影光效，甚至连访问计数器我也想过，也想过用各种小动物，小花小草，或者是民大的一些独特的元素来装饰我的网页。之后在html的学习中，这些创意大部分都没有实现，我再一次看了我README的源代码，没想到在一个小小特效的背后是几十行甚至上百行的代码，而我需要一行一行的去学习和了解其意思，实话说目前学到并运用到我的网页中的只是我构想中的冰山一角。

## 项目进行时

* html：我先确定了我要展示的内容，每段写什么，放不放图片等，之后确定好用哪个标签和元素，输入内容，插入超链接（再加一个小奖牌当小创意）
* css：网页框架基本完成之后，我开始从字体大小，样式，颜色，居中，背景颜色等方面利用所学尽量丰富网页

## 最大难题

虽然已经学习了一部分html，css等的内容，但是由于没有应用过，很多语法都只是有大概的对于功能的印象，要使用了还要去搜索具体语法，非常浪费时间，之后我想到了先做html学习笔记，在笔记上整理各种常用语法，再巩固一次对于常用语法的用法

## 优化和补完

包是要优化和补完的，在制作期间常常让我感到力不从心，我脑海里构想出来的东西无论是去搜教程还是查找语法，始终无法去表现它，就算表现出来也是差强人意，在项目的构想我提到我的构想有大半都没有被实现，如果将来我对这方面有了更深层次的理解并且有了过硬的基础，我一定会回到最初的这个项目上，回到这第一个网页，用无数方法让我的起点变得美轮美奂，我一直幻想着每当我回头看看这个起点时，它会变得一次比一次美丽，也会给我更加强大的力量和坚定走向远方的自信

## 浅浅说说心里话

### 感受

在这十天的历程里我受益匪浅，我听了很多次翁恺老师的圣经：”学计算机一定要有一个非常强大的心理状态，计算机的所有东西都是人做出来的，别人想的出来，我也一定想的出来。在计算机里头没有任何黑魔法。计算机所有东西只是我现在还不知道而已，总有一天我会把所有细节搞明白。“这段话一次次的激励着我不断挖掘不断探索，就像我上文提到的，当一个小小特效的源代码铺天盖地冲进我的眼球时，我也曾迷茫，也曾惆怅，密密麻麻的代码各有各的功能，我也尝试过将我的github主页源代码进行一些删减和简化，却发现，这些代码之间有着千丝万缕的联系，随便删除几条就可能导致整体无法渲染出来。我不由得佩服这串代码的原作者，他当时在对这些代码进行嵌套的时候，他的思维到底是什么样的，我能清晰感觉到他思维的复杂程度，复杂到我根本找不到这串代码的主体或者说是躯干，它给我的感觉就像一颗摸不到树干的大树，我的身边早已枝繁叶茂，所以我才找不到树干。我充分意识到自己无论是逻辑思维还是代码基础都还有很长的路要走，只能带着翁恺老师的话和geek的内涵一遍遍摸索，我相信再细小的枝叶存在主干，只是等待我拨开叶穗去发现罢了。就像之前吴哥夸我个人主页很炫酷，我回答他，都是找来拼拼凑凑的，总有一天我会自己写出这些东西，总有一天我会有极大的改变之力！

### 不足

之前和考核组的其他几位同学交流的时候看到了他们制作的网页，都非常精美，我感觉到自己所制作的网页是如此小儿科，我连如何把图片三张并排放到一起都做不出来，有了使用AI或者去网上搜索源代码的想法，可是看到考核任务书上尽力而为的字样，并且是第一次制作网页，也不想欺骗自己，最后还是选择手搓，结果就是收获了一个丑丑的网页和一大堆感受。在制作过程中，我在使用css进行丰富的时候浪费了很多时间。

1. 第一个问题就是嵌套问题，我的代码可以说是一排排铺着下去的，并且是单行的，在我尝试用<div>把各种东西包装进去的时候，我总是分不清楚什么东西在内什么东西在外，该把那一个个class放在哪里，所以在嵌套的思路上有待提高（勤加练习）；
2. 第二个问题是分层归类问题，在我的网页源代码中，只分了body和footer两部分，footer还是最后为了展示分层特意加上去的，并且不会归类，在对比其他源代码和我的源代码时，总感觉我的代码有很强烈的违和感，可能原因就在这里；
3. 第三个问题是目前写代码的方式的问题，在网页制作过程中，我可能更偏向于通过功能来查找进行制作，而没有经过系统性的学习，有些内容是我没有学到但是通过查找运用进去的，在之后的学习制作过程中还是要重视系统性的学习，在脑子里形成网络，把功能和代码联系起来并进行分类可能更有助于今后的制作
